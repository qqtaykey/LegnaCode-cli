# LegnaCode — AI 编程助手与可视化协作工作空间

## 核心痛点

AI 编程助手存在两个根本问题：**黑盒不透明**和**多 Agent 协作不可控**。

开发者启动 AI 助手后只能盯终端等输出，不知道 Agent 在干什么、调了哪些工具、卡在哪步。多 Agent 并行时更严重——谁在写代码、谁在跑测试、谁在等权限，全靠翻日志猜。信息不对称导致信任度低、干预滞后、协作效率打折。

LegnaCode 的解法：**把 AI 工作过程变成像素风办公室，每个 Agent 是动画角色，所有状态实时可见。**

## 核心逻辑流

### 长链推理

CLI 基于大语言模型驱动，支持完整长链推理闭环：

```
用户指令 → 模型推理 → 工具调用（Read/Edit/Bash/Grep...）→ 结果回传 → 继续推理 → 循环至完成
```

单次任务可能数十轮工具调用，模型每轮根据上下文决定下一步。会话 JSONL 持久化，支持断点续接。

### 多 Agent 协作

复杂任务通过 Agent Teams 分解：

```
Lead Agent → 拆分子任务 → 启动 Teammate #1、#2（独立终端/独立上下文）→ 并行执行 → 汇总交付
```

每个 Agent 独立运行，Lead 通过 TaskList 协调。真正的多进程并行，非串行模拟。

### 实时事件链路（核心差异化）

Agent 内部状态实时投射到像素办公室：

```
CLI 执行工具 → HTTP POST 端口 19960 → Office Server 路由事件 → 状态映射 → Canvas 实时渲染
```

事件类型：SessionStart（角色入场）、PreToolUse（打字动画+工具气泡）、PostToolUse（气泡消失）、Stop（角色离场）。JSONL 轮询作为 fallback 双通道保证可靠性。

### 角色状态机

每个像素角色独立运行：TYPE（打字+工具名气泡）、IDLE（随机漫步）、WALK（A*寻路）、权限气泡（等待审批）、矩阵特效（上下线动画）。办公室座位满时自动扩容。

## 技术栈

Bun 编译原生二进制，React + Canvas 2D 像素引擎，HTTP + postMessage 双通道，跨 5 平台（macOS/Linux/Windows），MIT 开源。
